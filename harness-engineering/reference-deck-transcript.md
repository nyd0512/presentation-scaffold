## Harness Engineering 고급편에 시사하는 점

이 참고 자료는 우리가 만들 **Harness Engineering (고급편)** 과 **메시지가 매우 가깝다**:

1. **"바깥을 먼저 손본다"** = 우리 README §3 위험 1 ("내 harness 투어")의 해독제 — 모델·프롬프트가 아닌 surrounding을 엔지니어링하는 게 본질
2. **"표지판 촘촘함"** = 우리 S5 Hooks + Permission engineering의 핵심 메타포 — 차용 가능
3. **5인 공통점 → 본인 사례** 의 흐름 = 우리도 동일 패턴 쓸 수 있음 (각 분야 실무자 인용 → 본인 harness 케이스스터디 → 일반화)
4. **"원샷 신화 = ZERO"** = 우리 S6 Operations의 failure injection 슬라이드 도입에 활용 가능
5. **plan mode 3단계** = 우리 S2 Engine Architecture의 disagreement-driven integration과 같은 패턴 (제출 → 승인 → 실행)
6. **허니 배저 비유** = 좋은 엔지니어링 스토리텔링 — Claude Code 강점을 한 단어로 압축. 우리 deck도 한 가지 강력한 메타포가 있으면 좋음 (예: "scaffold", "표지판", "compass" 등)
