Tier 5 — 메모리 · 스토리지

밸류체인 최하단. 누가 이기든 메모리는 반드시 필요하다. AI 서버 원가의 50~65%를 차지.

HBM
High Bandwidth Memory · 휘발성
DRAM 다이를 수직으로 8~16층 적층하여 대역폭을 극대화한 특수 메모리. GPU 패키지 위에 직접 올라간다.
역할 — GPU 연산에 필요한 데이터를 초고속으로 공급. AI 학습·추론 속도를 결정하는 최대 병목.
웨이퍼 소모 일반 DRAM의 3~4배
DDR5
범용 DRAM · 휘발성
서버의 시스템 메모리. CPU가 사용하는 주 작업 공간이자 에이전트 세션 관리, KV-cache 오프로드에 활용.
역할 — AI 서버 1대에 2TB 필요. HBM과 별개로 반드시 탑재되는 부품. 에이전트 시대에 수요 급증.
Q1 2026 가격 QoQ +90~95%
NAND Flash · SSD
비휘발성 스토리지
전원이 꺼져도 데이터가 유지되는 저장 매체. 학습 데이터셋, 모델 파라미터, 체크포인트, 로그를 저장한다.
역할 — 수조 토큰의 학습 데이터와 수백GB 모델을 저장. 에이전트가 파일을 읽고 쓸 때마다 SSD를 경유.
Q2 2026 가격 QoQ +70~75%
DDR4
레거시 DRAM · 휘발성
이전 세대 시스템 메모리. 레거시 서버, 임베디드, 산업용, 자동차에서 여전히 사용. 생산 라인이 축소 중.
역할 — AI와 직접 무관하나, 같은 팹에서 웨이퍼를 공유. HBM 확대 → DDR4 공급 감소 → 동반 폭등.

HBM · DDR5 · NAND는 경쟁이 아니라 보완 관계

AI 서버 1대에 세 가지가 전부 들어간다. HBM은 GPU 옆에서 연산 데이터를 초고속 공급하고, DDR5는 CPU에서 시스템 작업을 처리하며, NAND SSD는 데이터와 모델을 저장한다. 셋 다 같은 팹에서 웨이퍼를 공유하므로, HBM 생산 확대 → 범용 DRAM/NAND 공급 감소 → 레거시 제품까지 동반 폭등하는 연쇄 구조가 작동한다.

에이전트 시대의 컴퓨팅 구조 변화

챗봇은 1회 응답에 수천 토큰을 소모한다. 에이전트는 계획 → 실행 → 검증 → 반복 루프를 자율적으로 돌리며, 한 작업에 수만~수십만 토큰을 사용하고 서브에이전트가 병렬로 작동한다. 같은 사용자 수에서 GPU·HBM·DDR5·SSD 수요가 한 자릿수 이상 증가하는 구조적 전환.
